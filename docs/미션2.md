# 💰 Python 파일 기반 콘솔 가계부 미션 가이드

> **"작은 서비스"**란 기능이 많은 게 아니라 예외 상황에서도 데이터가 안전한 것을 말합니다. 단순한 프로그램이 아니라, 유지보수 가능한 설계로 완성한다는 관점으로 접근하세요.

---

## 📌 1. 과제 개요 & 목표

이번 미션은 Python 표준 라이브러리만을 사용하여 **파일 입출력 기반의 가계부 콘솔 프로그램**을 구현하는 과제입니다. 수입/지출 내역의 단순 저장을 넘어 수정/삭제, 검색, 월별 요약, 예산 관리, 데이터 백업 및 복구까지 포함한 완성도 높은 서비스를 타겟으로 합니다.

### 🎯 핵심 배움 포인트
* **영구 저장 구조:** 파일 기반 저장(JSONL 또는 CSV)으로 데이터를 안전하게 관리하고 CRUD를 구현합니다.
* **계층형 아키텍처:** CLI / 서비스 / 저장소(파일 I/O) / 모델(데이터 구조)로 책임을 명확히 나눕니다.
* **스트리밍 처리:** `yield` 기반 제너레이터로 대용량 파일도 메모리 낭비 없이 처리합니다.
* **공통 관심사 분리:** 데코레이터(Decorator)를 활용해 로깅, 예외 처리, 시간 측정을 분리합니다.
* **정적 타이핑:** 타입 힌트(Type Hinting)를 적용해 함수와 데이터 구조 간의 계약을 명확히 합니다.

## 🛠️ 2. 개발 환경 & 제약 사항

* **언어 버전:** Python 3.10 이상
* **라이브러리:** **표준 라이브러리만 사용 가능** (`pip install`을 통한 외부 라이브러리 사용 금지)
* **저장 방식:** JSONL 또는 CSV 중 1개 선택 (3개 이상의 파일로 분리 저장 필수)
  * `transactions.<fmt>` (거래 내역)
  * `categories.<fmt>` (카테고리 관리)
  * `budgets.<fmt>` (월별 예산)
  * *기본 경로:* `./data/` 디렉토리 (옵션 `--data-dir`로 변경 가능해야 함)
* **CLI 규칙:** 옵션 표기는 리눅스 표준인 `--` 기호로 통일 (예: `--help`, `--limit`, `--from`)
* **오류 처리:** * 스택트레이스(Stacktrace) 노출 금지 ❌
  * **[원인 + 해결 힌트]** 형태로 UX 개선 출력 ⭕
  * 오류 종료 시 `exit code`는 `0`이 아닌 값으로 설정

## 📋 3. 기능 요구사항 (체크리스트)

### 1) 실행 및 입력 방식
- [ ] 실행 명령어 포맷 지원: `python -m budget_app <command> [options]`
- [ ] 모든 명령에서 `--help` 옵션 지원
- [ ] 데이터 입력은 기본적으로 **대화형(`input()`)** 방식 적용
- [ ] `search`, `list`, `summary`, `export`, `import`, `delete` 명령어는 옵션 인자 방식 지원

### 2) 데이터 모델 & 입력 검증
- [ ] 최소 2개 이상의 클래스 정의 (`dataclass` 권장)
- [ ] **Transaction 필수 필드:** `id`(유일값), `type`(income/expense), `date`(YYYY-MM-DD), `amount`(양수 정수), `category`, `memo`(선택), `tags`(선택)
- [ ] **입력 검증:** 날짜 형식 오류, 음수/0 금액, 잘못된 타입, 존재하지 않는 카테고리 진입 시 재입력 요구 또는 에러 메시지 출력

### 3) 주요 명령어 기능
- [ ] **`add` (거래 추가):** 대화형 입력, 카테고리 사전 검증, 완료 후 생성된 고유 ID 출력
- [ ] **`list` (목록 조회):** 최신순 정렬, `--limit N` 옵션 제공, **제너레이터 기반 스트리밍 필수**
- [ ] **`update` (거래 수정):** 문서에 **[옵션 방식]** 또는 **[대화형 방식]** 중 하나를 고정하여 구현 (전체 재작성/임시 파일/원자적 교체 등 데이터 안정성 고려)
- [ ] **`delete` (거래 삭제):** `delete --id <id>` 형태로 동작, 없는 ID 예외 처리
- [ ] **`search` (검색):** 필터 조건인 `--from`, `--to`, `--category`, `--type`, `--q`(메모 검색), `--tag` 지원 (최신순 및 스트리밍 유지)
- [ ] **`summary` (월별 요약):** 특정 월(--month)의 총수입, 총지출, 잔액 및 지출 TOP N(`--top N`) 출력. 데이터가 없는 달은 "데이터 없음" 명시
- [ ] **`budget` (예산 설정):** `budget set --month YYYY-MM --amount <금액>` 저장. `summary` 호출 시 예산 대비 사용률(%) 및 초과 경고 알림 연동
- [ ] **`category` (카테고리 관리):** `add/list/remove` 기능 제공. 삭제 시 해당 카테고리를 사용 중인 내역이 있다면 삭제를 막거나 대체 카테고리 지정 요구
- [ ] **`import / export` (데이터 입출력):** 정해진 CSV 스키마 표준을 따르며 UTF-8 및 헤더 포함 필수

#### 📊 가져오기/내보내기 CSV 스키마 사양
| Column | Required | Description |
| :--- | :---: | :--- |
| **date** | Y | YYYY-MM-DD 형식 |
| **type** | Y | `income` 또는 `expense` |
| **category** | Y | 시스템에 등록된 카테고리 |
| **amount** | Y | 양수 정수 |
| **memo** | N | 문자열 데이터 |
| **tags** | N | 쉼표(,)로 구분된 태그 문자열 |
